# 带AI功能的个人笔记应用PRD文档

## 1. 文档概述

### 1.1 文档背景
在信息爆炸的时代，学生、职场人士及知识工作者面临着信息过载、知识碎片化、内容创作效率低下等核心痛点。传统的笔记工具仅提供基础的记录与存储功能，缺乏对内容的智能理解、组织与再创作能力，导致用户花费大量时间在整理、查找和润色笔记上，难以将信息有效转化为知识资产。本项目旨在开发一款集成先进AI能力的下一代个人笔记应用，通过智能技术提升用户从记录到应用的全流程效率，解决“记了难找、记了难用”的核心问题。

### 1.2 项目目标
#### 业务目标 (上线后12个月内)
1.  **用户增长**：实现累计注册用户达100万，月活跃用户(MAU)达30万。
2.  **用户留存**：次日留存率不低于40%，第30日留存率不低于15%。
3.  **用户满意度**：通过NPS（净推荐值）或应用商店评分衡量，用户满意度达到85%以上。
4.  **商业化探索**：为后续高级AI功能订阅服务奠定用户基础，探索付费转化路径。

#### 产品目标
1.  **功能完备**：在6个月开发周期内，完成核心的笔记创建、编辑、管理及三项AI功能的开发与上线。
2.  **体验优异**：核心操作路径（创建、编辑、搜索）用户任务完成率达95%以上，AI功能平均响应时间低于2秒。
3.  **安全可靠**：实现本地数据加密与安全的端到端同步，确保用户隐私，上线后无重大（P0级）安全漏洞报告。

#### 用户价值
1.  **效率提升**：通过AI智能整理与内容辅助，将用户从繁琐的笔记归类、写作构思中解放出来，提升信息处理效率。
2.  **知识沉淀**：通过智能标签、摘要和关联推荐，帮助用户构建结构化的个人知识体系，便于回顾与深化理解。
3.  **灵感激发**：在写作瓶颈时提供灵感扩展和建议，辅助用户更好地进行内容创作。

### 1.3 受众范围
-   **产品与设计团队**：明确产品需求、设计方向和交互细节。
-   **开发团队（前端、后端、移动端、AI算法）**：理解功能逻辑、技术架构和实现细节。
-   **测试团队**：依据本文档编写测试用例，进行功能与性能验证。
-   **运营与市场团队**：了解产品核心卖点与用户价值，为上线推广和后续运营做准备。
-   **项目管理者**：用于项目进度把控、资源协调和里程碑评审。

## 2. 业务说明

### 2.1 用户角色
| 角色 | 身份/特征 | 核心诉求 |
| :--- | :--- | :--- |
| **学生** | 中学生、大学生、研究生 | 快速记录课堂要点、整理复习资料、生成论文大纲、高效管理阅读笔记。 |
| **职场人士** | 白领、管理者、专业人士 | 记录会议纪要、整理项目资料、撰写工作报告、管理待办事项、积累行业知识。 |
| **知识工作者** | 作家、记者、研究员、创作者 | 收集创作素材、记录灵感瞬间、梳理逻辑框架、进行深度写作、构建知识网络。 |

### 2.2 业务流程
**正常流程：用户完成一次从记录到回顾的知识闭环**
1.  **创建/编辑笔记**：用户通过文字、语音（需转文本）、图片（需OCR）或剪藏网页等方式创建或编辑一条笔记。
2.  **AI智能整理（自动触发）**：笔记保存时，系统自动在后台调用AI服务，分析内容，生成**建议标签**和**内容摘要**，并推荐1-3个**分类文件夹**。用户可一键确认或修改后归档。
3.  **AI内容辅助（手动触发）**：在编辑笔记时，用户可选中部分文本或点击“AI助手”按钮，请求**写作建议**、**语法修正**或**灵感扩展**。AI返回结果后，用户可选择采纳、修改后插入或忽略。
4.  **智能检索**：用户在搜索框内输入自然语言问题（如：“上周开会关于产品定价的结论”），系统理解语义，从所有笔记中检索相关内容，并按相关性排序展示。同时，在查看单条笔记时，侧边栏会显示“相关笔记”推荐。
5.  **同步与多端使用**：用户在一台设备上的所有操作（笔记内容、标签、分类），通过端到端加密的方式，安全地同步到其账号下的其他设备（iOS/Android）。

**异常流程：**
1.  **网络异常**：
    -   **离线创建/编辑**：所有功能均可在离线状态下使用，数据本地保存。AI功能（除基于本地轻量模型的简单修正外）需提示“网络连接后可用”。新数据将加入同步队列。
    -   **同步冲突**：当多设备离线修改同一笔记后上线，检测到冲突时，应用应提示用户，并提供“查看差异并手动合并”或“以最新版本/指定设备版本为准”的选项。
2.  **AI处理异常**：
    -   **内容无法理解/过短**：当笔记内容过短或无意义时，AI整理功能不生成建议，或仅生成通用标签（如“未分类”）。
    -   **AI服务超时或失败**：前端显示“AI处理中，请稍后”状态，若超过3秒未响应，则降级为保存笔记内容，并Toast提示“AI服务暂不可用，笔记已保存”，后续可在笔记详情页手动触发AI整理。

### 2.3 使用场景
**场景一：学生整理课堂笔记**
> 小王在哲学课上用手机快速记录了大量零散观点。下课后，他打开应用，系统已自动为这条长笔记生成了“西方哲学”、“柏拉图”、“理念论”等标签和一个简明摘要。小王一键将其归入“大学课程/哲学史”文件夹。一周后复习时，他搜索“柏拉图关于国家的观点”，立刻找到了这条笔记和相关联的阅读材料笔记。

**场景二：职场人士撰写项目报告**
> 李经理需要撰写季度项目复盘报告。他在应用中新建文档，写下几个要点后思路卡壳。他选中一段文字，点击“灵感扩展”，AI提供了几个可行的分析维度和表述方式。在撰写过程中，语法修正功能实时提示了几处冗长句子的修改建议。报告完成后，AI自动生成的摘要帮助他快速核对了核心内容。

**场景三：知识工作者进行主题研究**
> 张研究员正在研究“气候变化对农业的影响”。他长期将相关论文摘要、新闻剪报、自己的思考片段都记录在该应用中。通过智能检索，他输入“找找之前关于极端天气和粮食产量的关联记录”，系统不仅找到了精确笔记，还在侧边栏推荐了他半年前记录的一条关于“病虫害模型”的笔记，为他提供了新的研究联想。

## 3. 功能需求

### 3.1 功能全景
应用整体功能架构分为四层：
1.  **核心笔记层**：提供基础的笔记创建、编辑、格式化、组织（文件夹/标签）与管理（删除、移动、分享）能力。
2.  **AI能力层**：集成三大核心AI功能模块，作为赋能上层应用的引擎。
3.  **功能应用层**：将AI能力封装成用户可感知和交互的具体功能点。
4.  **系统支撑层**：包括用户账户、数据同步、安全加密、设置等后台支撑系统。

```
用户界面
├── 笔记列表页 (首页)
├── 笔记编辑器页
├── 笔记本/标签管理页
└── 搜索与发现页

功能应用层
├── 【智能整理】自动标签/摘要/分类推荐
├── 【内容辅助】写作建议/语法修正/灵感扩展
└── 【智能检索】自然语言搜索/关联推荐

AI能力层 (后台服务)
├── NLP理解模型 (用于摘要、标签、分类)
├── 文本生成模型 (用于写作辅助、灵感扩展)
└── 语义检索模型 (用于搜索与关联)

核心笔记层
├── 富文本编辑
├── 笔记本/标签体系
├── 附件管理 (图片、文件)
└── 基础操作 (增删改查、分享)

系统支撑层
├── 用户认证与账户
├── 端到端加密同步
├── 本地数据存储
└── 应用设置
```

### 3.2 详细功能列表

#### 3.2.1 功能模块：笔记管理

| **编辑与格式化** | 提供基础的文本格式设置（加粗、斜体、标题等）和内容编辑功能。 | 在编辑器页，通过顶部或选中文本弹出的工具栏进行操作。 | 遵循常见的富文本编辑交互规范。 |

#### 3.2.2 功能模块：智能整理 (AI-Powered Organization)
| 功能名称 | 功能说明 | 操作流程 | 输入/输出规则 |
| :--- | :--- | :--- | :--- |
| **自动生成标签** | AI分析笔记全文，提取3-5个最具代表性的关键词作为标签建议。 | **自动**：笔记保存/内容发生重大变更后触发。<br>**手动**：在笔记详情页点击“AI整理”按钮。 | **输入**：笔记文本内容。<br>**输出**：3-5个标签建议气泡，用户可“一键添加全部”、“选择性添加”或“忽略”。生成的标签自动加入用户标签库。 |

| **智能分类推荐** | AI根据笔记内容，推荐1-3个最匹配的现有笔记本（文件夹）供用户快速归档。 | 同“自动生成标签”触发时机。 | **输出**：推荐列表显示在保存按钮附近或浮动面板。用户点击即可完成移动。 |

#### 3.2.3 功能模块：内容辅助 (AI Writing Assistant)
| 功能名称 | 功能说明 | 操作流程 | 输入/输出规则 |
| :--- | :--- | :--- | :--- |
| **写作建议** | 根据上下文，对选中的段落或全文提供结构优化、语气调整、逻辑增强等建议。 | 1. 选中文本或光标置于段落中。<br>2. 点击弹出工具栏的“AI助手”图标。<br>3. 选择“改进写作”。 | **输出**：提供1-3条具体的修改建议文本。用户点击某条建议可预览替换效果，并选择“替换”或“取消”。 |
| **语法修正** | 检查并修正文本中的拼写错误、语法错误和标点误用。 | 1. 同“写作建议”步骤1-2。<br>2. 选择“检查语法”。 | **输出**：以高亮下划线（如红色波浪线）标出疑似问题处。点击下划线显示修正建议，可一键修正全部或逐个处理。**（假设：此功能可集成轻量本地模型，实现离线基础修正）** |

#### 3.2.4 功能模块：智能检索 (Smart Search)
| 功能名称 | 功能说明 | 操作流程 | 输入/输出规则 |
| :--- | :--- | :--- | :--- |

| **关联笔记推荐** | 在阅读单条笔记时，系统在侧边栏或底部推荐与该笔记内容相关的其他笔记。 | **自动**：打开笔记详情页后自动加载。<br>**手动**：可点击“查看更多关联”。 | **输出**：显示“相关笔记”板块，包含3-5条笔记的标题和摘要预览。点击可跳转。 |

### 3.3 页面说明 (核心页面)
**页面一：笔记列表页 (首页)**
-   **元素**：顶部搜索栏、笔记本/标签筛选器、笔记列表（卡片式展示笔记标题、摘要/首行、标签、日期）、底部导航栏（首页、搜索、新建按钮、个人中心）。
-   **交互**：下拉刷新同步，上拉加载更多。左滑笔记卡片可快速“删除”、“添加标签”。点击卡片进入编辑页。长按进入多选模式。

**页面二：笔记编辑器页**
-   **元素**：顶部返回按钮、标题输入框、AI助手浮动按钮、格式工具栏、正文编辑区、底部键盘/扩展工具栏（插入图片、待办等）。
-   **交互**：输入时自动保存草稿。点击AI助手按钮，根据是否选中文本，弹出功能菜单（写作建议/语法修正/灵感扩展）。保存时，触发智能整理，结果以非模态面板（从底部弹出）形式展示。

**页面三：搜索页**
-   **元素**：显眼的搜索输入框、历史搜索记录、热门标签快捷入口、实时搜索结果列表。
-   **交互**：输入时实时显示搜索结果。支持清空历史记录。

### 3.4 权限说明
-   **所有注册用户**：拥有全部笔记管理、AI功能使用、数据本地操作权限。
-   **数据权限**：所有笔记数据归属创建者本人。通过端到端加密，服务器无法解密内容。分享功能（假设为后续迭代）将生成加密链接，由创建者设置密码或有效期。
-   **功能权限**：无差异化功能权限。所有上线功能对所有用户开放。（高级AI模型调用次数限制可作为未来订阅服务的权限点，不在V1.0范围）。

## 4. 非功能性需求

### 4.1 性能需求
-   **应用启动时间**：冷启动时间不超过2秒。
-   **页面响应**：列表页加载、笔记打开时间不超过1秒。
-   **AI响应时间**：在良好网络条件下，AI功能（智能整理、内容辅助、搜索）的响应时间**95%的请求应在2秒内完成**，最大不超过5秒。
-   **同步性能**：增量同步应在网络连接后30秒内完成。
-   **离线性能**：所有非依赖网络的本地操作（编辑、查看、本地搜索）应流畅无卡顿。

### 4.2 兼容性需求
-   **iOS**：支持iOS 15及以上版本，适配iPhone主流机型，特别关注全面屏适配。
-   **Android**：支持Android 10及以上版本，适配主流品牌机型（华为、小米、OPPO、Vivo等）及不同屏幕尺寸。
-   **后续Web端**：计划支持Chrome、Safari、Edge、Firefox最新两个稳定版本。

### 4.3 安全性需求
-   **数据加密**：本地数据库（SQLite/Realm）必须使用强加密算法（如SQLCipher）进行加密。同步过程中的数据传输使用TLS 1.3。
-   **端到端加密**：笔记内容、标题、标签在客户端使用用户独有的密钥加密后，再上传至服务器。服务器仅存储加密后的密文。密钥由用户设备生成和管理，**不上传至服务器**。
-   **隐私保护**：严格遵守“严禁收集用户隐私数据”约束。AI模型训练**不得使用用户笔记内容**。必要的匿名化使用数据（如功能点击率、性能指标）需经用户同意，并明确告知用途。
-   **内容标识**：所有AI生成或修改的内容（如采纳的写作建议、生成的摘要），需在界面上有视觉区分（如浅灰色背景、小号字体）或通过“AI生成”图标进行标注。
-   **防攻击**：具备基础的防暴力破解、防注入攻击能力。

### 4.4 可用性需求
-   **系统可用性**：核心服务（同步、AI接口）月度可用性不低于99.5%。
-   **容错能力**：网络异常时给出明确友好提示。应用崩溃率低于0.1%。
-   **异常提示**：所有用户操作失败或系统异常，均需通过Toast、Snackbar或对话框等形式，给出明确、友好、可指导下一步操作的提示信息。
-   **无障碍**：遵循WCAG 2.1 AA级标准，支持屏幕阅读器（VoiceOver/TalkBack）基本访问。
