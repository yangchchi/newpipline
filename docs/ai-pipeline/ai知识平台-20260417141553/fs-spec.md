# FS

## 1. 目标
解决公司内部AI资源分散、知识无法沉淀、需求对接困难的问题，打造一站式AI资源与协作平台。成功标准：上线6个月内，平台月活跃用户（MAU）达到公司员工总数的40%。

## 2. 角色与场景
- **用户**：公司全体员工，主要角色：浏览者/学习者、需求方、AI专家/供给方、内容贡献者、平台运营者、系统管理员。
- **使用场景**：
    1.  员工寻找AI工具解决工作问题。
    2.  员工学习AI知识或查看实践案例。
    3.  业务方发布AI需求，技术专家接单协作。
    4.  专家分享AI实践案例。

## 3. 用户故事
### 用户故事1：作为浏览者，我希望通过自然语言提问，快速获得基于公司内部知识的准确答案。
#### 验收标准：
##### (1) 用户在智能问答台界面输入问题文本，点击“发送”。
##### (2) 系统调用大模型API，结合知识库检索结果，流式生成答案。
##### (3) 答案末尾显示引用的源文档名称（可点击查看原文）。
##### (4) 答案下方提供“有帮助”和“无帮助”反馈按钮。

### 用户故事2：作为需求方，我希望发布一个AI需求，以便找到专家解决我的问题。
#### 验收标准：
##### (1) 用户登录后，在需求广场点击“发布需求”。
##### (2) 用户填写必填字段：标题、详细描述（含背景、目标、期望产出）、需求分类、期望完成时间。
##### (3) 用户点击“发布”，需求状态变为“待响应”，并出现在需求广场列表中。
##### (4) 系统向标签匹配的AI专家推送新需求通知。

### 用户故事3：作为AI专家，我希望接单一个感兴趣的需求，以便提供服务并获得回报。
#### 验收标准：
##### (1) AI专家在需求广场浏览“待响应”的需求列表。
##### (2) 专家点击某个需求进入详情页，查看完整信息。
##### (3) 专家点击“接单”按钮。
##### (4) 需求方收到接单通知，并在其“我的需求”中看到该需求状态变为“待确认”，且接单者列表中出现该专家。

### 用户故事4：作为内容贡献者，我希望分享一个AI应用案例，以便帮助他人并展示能力。
#### 验收标准：
##### (1) 用户登录后，在最佳实践区点击“分享案例”。
##### (2) 用户填写案例表单：标题、应用场景、所属分类、使用工具、详细步骤、心得总结。
##### (3) 用户提交后，案例状态变为“待审核”，并进入运营后台审核队列。
##### (4) 运营人员审核通过后，案例在最佳实践区公开显示。

## 4. 规则补充
### 全局业务规则
1.  **权限规则**：
    -   IF 用户未登录 THEN 仅可浏览公开内容，不能发布需求、接单、发布案例。
    -   IF 用户角色为“需求方” THEN 可发布需求。
    -   IF 用户角色为“AI专家” THEN 可接单。
    -   IF 用户角色为“运营者” THEN 可审核内容。
    -   IF 用户角色为“管理员” THEN 可管理用户与权限。

2.  **需求广场-接单规则**：
    -   IF 需求状态为“待响应” THEN 多个AI专家可点击“接单”。
    -   IF 需求方从接单者列表中选定一人并确认 THEN 需求状态变为“进行中”，其他接单者自动落选。
    -   IF 需求状态不为“待响应” THEN “接单”按钮置灰或隐藏。

3.  **智能问答台-降级规则**：
    -   IF 调用大模型API超时（>5秒）或返回错误 THEN 系统降级为仅返回知识库检索到的相关文档片段，并在答案开头显示提示：“以下信息基于知识库检索，仅供参考”。
    -   IF 知识库检索结果为空 THEN 返回提示：“未在知识库中找到相关信息”。

4.  **状态流转规则（需求广场）**：
    -   IF 新需求发布成功 THEN 状态 = “待响应”。
    -   IF 需求被接单且需求方确认 THEN 状态 = “进行中”。
    -   IF 供给方提交交付物 THEN 状态 = “待验收”。
    -   IF 需求方确认验收 THEN 状态 = “已完成”。
    -   IF 需求方在状态为“待响应”时取消需求 THEN 状态 = “已关闭”。
    -   IF 需求方在状态为“进行中”时取消需求 THEN 需触发协商流程，经供给方同意后状态 = “已关闭”。

### 功能规格：智能问答台-自然语言问答
-   **输入**：用户输入的纯文本问题。
-   **规则**：
    1.  IF 用户点击“发送” THEN 调用问答接口，传入问题文本。
    2.  IF 接口调用成功 THEN 以流式（SSE）方式在界面逐字输出答案。
    3.  IF 答案生成完成 THEN 在答案末尾追加一行“来源：[文档1名称](链接)，[文档2名称](链接)”。
    4.  IF 用户点击“有帮助”/“无帮助” THEN 记录反馈数据。
-   **输出**：格式化的答案文本流。支持基础Markdown渲染（加粗、列表、代码块）。末尾附来源。
-   **正常示例**：
    -   输入：“公司年假有多少天？”
    -   输出：“根据《员工手册2024版》规定，公司年假天数根据工龄计算... 来源：《员工手册2024版》第5章”
-   **异常示例**：
    -   输入：“如何造火箭？”
    -   输出：（降级后）“未在知识库中找到相关信息。”

### 功能规格：需求广场-需求发布
-   **输入**：表单字段：标题（文本，≤50字）、详细描述（文本，≤1000字）、需求分类（单选）、期望完成时间（日期）。
-   **规则**：
    1.  IF 用户点击“发布” THEN 校验所有必填字段非空。
    2.  IF 校验通过 THEN 调用发布接口，创建需求记录，状态置为“待响应”。
    3.  IF 校验不通过 THEN 在对应字段下方显示红色错误提示“该字段为必填项”。
-   **输出**：创建成功，跳转到该需求详情页；创建失败，停留在发布页并显示错误。
-   **正常示例**：
    -   输入：标题=“自动生成周报PPT脚本”， 详细描述=“每周从A、B系统导出数据...”， 分类=“自动化”， 期望完成时间=“2024-06-30”
    -   输出：跳转到新需求详情页，状态显示“待响应”。
-   **异常示例**：
    -   输入：标题=“”， 详细描述=“需要帮助”， 分类=未选， 期望完成时间=“2024-05-01”（过去日期）
    -   输出：提示“标题为必填项”、“分类为必填项”、“期望完成时间不能早于今天”。

### 功能规格：需求广场-需求接单
-   **输入**：需求ID，接单者用户ID。
-   **规则**：
    1.  IF 用户点击“接单”按钮 THEN 校验需求当前状态是否为“待响应”。
    2.  IF 校验通过 THEN 调用接单接口，将当前用户加入该需求的接单者列表。
    3.  IF 校验不通过（如状态已非“待响应”） THEN 前端toast提示“该需求已无法接单”。
-   **输出**：接单成功，按钮变为“已接单”，需求方收到通知；接单失败，显示错误提示。
-   **正常示例**：
    -   输入：需求ID=123， 接单者ID=user_ai_exp
    -   输出：接单成功，需求详情页的接单者列表显示user_ai_exp。
-   **异常示例**：
    -   输入：需求ID=456（状态已为“进行中”）， 接单者ID=user_ai_exp
    -   输出：前端提示“该需求已无法接单”。

### 功能规格：最佳实践区-案例发布
-   **输入**：表单字段：标题（文本）、场景（文本）、分类（单选）、使用工具（多选）、详细步骤（富文本）、心得（文本）。
-   **规则**：
    1.  IF 用户提交案例 THEN 所有必填字段非空校验。
    2.  IF 校验通过 THEN 调用创建接口，案例状态置为“待审核”，并进入运营后台列表。
    3.  IF 运营人员在后台点击“审核通过” THEN 案例状态置为“已发布”，前端列表可见。
    4.  IF 运营人员点击“审核驳回” THEN 需填写驳回理由，案例状态置为“已驳回”，创建者可见。
-   **输出**：提交成功，提示“案例已提交审核，请耐心等待”；提交失败，显示具体错误。
-   **正常示例**：
    -   输入：标题=“使用Midjourney生成电商海报”， 场景=“营销活动”， ...
    -   输出：提示“案例已提交审核，请耐心等待”。
-   **异常示例**：
    -   输入：标题=“”， 场景=“”， ...
    -   输出：提示“标题和场景为必填项”。
