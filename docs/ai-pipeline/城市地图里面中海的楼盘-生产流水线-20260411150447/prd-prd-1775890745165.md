## 1. 文档概述

### 1.1 文档背景
*   **业务痛点**：目前，投研团队在进行城市市场分析、项目价值评估时，需在多套系统（CRM、Excel报表、第三方地图）间切换，以定位和查询“中海地产”旗下楼盘信息。此过程耗时、低效，且无法直观地将楼盘地理分布与周边配套、市场热力图等信息结合分析，影响研判效率和决策准确性。
*   **核心问题**：缺乏一个集成的、可视化的工具，帮助投研人员快速在地理空间维度上，定位、筛选、查看目标楼盘的核心属性，进行初步价值评估。

### 1.2 项目目标
*   **业务目标 (SMART原则)**：
    *   **可量化**：上线后1个月内，投研团队使用地图进行楼盘定位和分析的日均时长提升30%。
    *   **可量化**：将单个楼盘的基础信息查询与定位平均耗时从目前的5分钟降低至30秒以内。
*   **产品目标**：
    *   在现有地图页成功集成“中海楼盘”数据图层，实现楼盘数据的可视化展示与交互。
    *   提供流畅的楼盘搜索、筛选、详情查看功能，提升投研工具链的完整性。
*   **用户价值**：为投研人员提供“一站式”的楼盘地理信息查询工具，提升市场扫描、竞品分析和项目初判的效率与体验。

### 1.3 受众范围
*   **产品/设计团队**：明确产品功能与交互细节。
*   **前端开发团队**：负责Web端地图集成、图层控件、前端交互逻辑实现。
*   **后端开发团队**：负责数据接口开发、数据同步逻辑。
*   **测试团队**：根据本文档编写测试用例，进行功能与兼容性测试。
*   **数据团队**：确保内部楼盘数据库的准确性与接口的实时性。
*   **业务方 (投研团队)**：确认需求与最终验收。

## 2. 业务说明

### 2.1 用户角色
| 角色 | 核心诉求 |
| :--- | :--- |
| **投研分析师** | 1. **快速定位**：能快速在地图上找到目标城市的中海楼盘。<br>2. **信息获取**：能直观看到楼盘的基础属性（位置、状态、户型）并查看详情。<br>3. **筛选聚焦**：能根据案名、状态等条件过滤，聚焦于特定类型的楼盘进行分析。 |
| **投研总监** | 1. **宏观概览**：能一眼看清区域内所有中海楼盘的分布密度与状态概况。<br>2. **高效决策**：能基于地图可视化信息，快速进行区域对比和项目初筛。 |

### 2.2 业务流程
*   **正常流程**：
    1.  用户登录系统，进入地图页面。
    2.  用户在地图控件区找到并点击“中海楼盘”图层开关，图层开启。
    3.  地图上自动渲染出符合当前地图视野范围的所有中海楼盘图标。
    4.  用户可通过：
        *   **浏览**：直接缩放、平移地图，查看不同区域的楼盘。
        *   **搜索**：在搜索框输入楼盘案名，地图定位并高亮显示该楼盘。
        *   **筛选**：在筛选器中选择“楼盘状态”（如“在售”），地图上仅显示状态为“在售”的楼盘。
    5.  用户点击任意楼盘图标，地图上弹出该楼盘的详细信息弹窗。
    6.  用户查看完信息后，点击弹窗关闭按钮或地图空白处，弹窗关闭。
    7.  用户分析完毕，可点击图层开关关闭“中海楼盘”图层。
*   **异常流程**：
    *   **无数据**：当前地图视野内无中海楼盘时，不显示任何图标，并在图层开关旁给出“当前区域无数据”的轻量提示。
    *   **数据加载失败**：调用数据接口失败时，图层开关置灰不可用，并显示Toast提示：“数据加载失败，请稍后重试”。
    *   **网络异常**：用户操作时网络断开，应提示“网络连接已断开”。

### 2.3 使用场景
1.  **场景一（区域市场扫描）**：投研分析师小王接到任务，需分析A市高新区房地产市场。他打开地图，开启“中海楼盘”图层，快速看到该区域有3个中海项目，其中2个“在售”，1个“售罄”，从而对中海在该区域的布局和去化情况有了直观印象。
2.  **场景二（特定项目查询）**：分析师小李在报告中需要引用“中海·寰宇天下”项目的具体信息。他直接在地图页搜索框输入“寰宇天下”，地图迅速定位到该项目并弹出详情窗，他快速复制了所需字段。
3.  **场景三（竞品对比准备）**：投研总监需要对比B市几个核心板块的在售项目。他开启图层后，使用筛选功能，只显示状态为“在售”的楼盘，地图上立刻只呈现了5个目标项目，方便他进行下一步的深入数据对比。

## 3. 功能需求

### 3.1 功能全景
本功能作为“投研地图平台”的一个新增数据图层模块，核心架构如下：
```
投研地图平台
├── 基础地图功能（缩放、平移、底图切换等）
├── **【新增】中海楼盘图层模块**
│   ├── 图层控制（开关）
│   ├── 楼盘数据可视化（图标渲染）
│   ├── 楼盘搜索功能
│   ├── 楼盘筛选功能（案名、状态）
│   └── 楼盘详情查看（信息弹窗）
└── 其他数据图层（如：土地出让、商圈规划等）
```

### 3.2 详细功能列表

#### 功能1：中海楼盘图层控制
*   **功能说明**：提供可视化开关，控制“中海楼盘”图层在地图上的显示与隐藏。
*   **操作流程**：
    1.  用户进入地图页，在右侧或上方的图层控制面板中，找到“中海楼盘”选项。
    2.  点击其旁边的“开关”按钮，按钮状态变为“开启”（视觉上可表现为打勾/高亮），地图上立即显示所有中海楼盘图标。
    3.  再次点击开关，按钮状态变为“关闭”，地图上所有中海楼盘图标消失。
*   **输入/输出规则**：
    *   输入：用户点击开关的动作。
    *   输出：开关状态变化；地图上对应图层的显示/隐藏。

#### 功能2：楼盘数据可视化与交互
*   **功能说明**：根据后端接口返回的楼盘坐标和基础数据，在地图对应位置渲染图标。图标样式需能区分不同楼盘状态，并支持点击交互。
*   **操作流程**：
    1.  图层开启后，前端调用接口，获取当前地图视野范围内的楼盘数据。
    2.  根据每个楼盘的`状态`字段，使用预设的图标样式进行渲染（例如：绿色-在售，灰色-售罄，黄色-在建）。
    3.  用户鼠标悬停在图标上时，图标轻微放大或颜色加深，并显示楼盘案名的Tooltip。
    4.  用户点击图标，触发“功能4：楼盘详情查看”。
*   **数据规则**：
    *   图标聚合：当地图缩放到较小级别（视野范围很大）时，若多个楼盘位置过于接近，可使用聚合点图标，点击聚合点可放大该区域或展开列表。
    *   **【假设】**：图标样式由设计团队根据现有地图风格定义，需包含“在售”、“待售”、“售罄”、“在建”四种状态的区分设计。

#### 功能3：楼盘搜索与筛选
*   **功能说明**：提供搜索框和下拉筛选器，帮助用户快速定位目标楼盘。
*   **操作流程**：
    *   **搜索**：
        1.  用户在图层控制面板或地图顶部的搜索框中，输入楼盘案名的全部或部分关键字。
        2.  系统实时调用接口进行模糊匹配，并在下拉列表中显示匹配结果（最多10条）。
        3.  用户点击列表中的一条结果，地图平滑移动并缩放到该楼盘位置，同时高亮该楼盘图标并自动弹出其详情弹窗。
    *   **筛选**：
        1.  用户点击“筛选”按钮，展开筛选面板。
        2.  用户可选择一个或多个“楼盘状态”（多选）。
        3.  点击“应用”后，地图上仅显示符合所选状态的楼盘图标，其他图标隐藏。
        4.  点击“重置”，清空筛选条件，显示所有楼盘。
*   **输入/输出规则**：
    *   搜索接口需支持对`案名`字段的模糊查询。
    *   筛选条件通过接口的`status`参数传递，支持多状态查询（如`status=在售,待售`）。

#### 功能4：楼盘详情查看
*   **功能说明**：用户点击楼盘图标后，弹出模态框展示该楼盘的详细信息。
*   **操作流程**：
    1.  用户点击地图上任一楼盘图标。
    2.  在该图标旁弹出固定大小的信息弹窗。
    3.  弹窗内以结构化形式展示楼盘信息。
    4.  用户可点击弹窗右上角的“X”或地图空白区域关闭弹窗。
*   **信息内容**：
    *   **【假设】** 根据《中海楼盘数据字典V1.2》，MVP版本弹窗至少包含以下字段：
        *   楼盘案名
        *   所属城市/区域
        *   楼盘状态
        *   主力户型（如：90㎡三房，120㎡四房）
        *   最新均价（若有）
        *   项目地址
        *   数据更新时间

### 3.3 页面说明
*   **核心页面**：现有地图页面 (`/map`)
*   **新增元素**：
    1.  **图层控制面板**：新增“中海楼盘”复选框及开关。
    2.  **搜索筛选栏**：集成于地图上方，包含一个搜索输入框和一个“筛选”按钮。
    3.  **楼盘图标**：覆盖在地图上的可点击图标，样式随状态变化。
    4.  **信息弹窗组件**：一个设计统一的浮动弹窗，用于展示楼盘详情。
*   **交互逻辑**：
    *   图层开关、搜索、筛选操作均会触发地图重新请求数据并渲染。
    *   弹窗出现时，应置于地图所有元素之上。点击弹窗内部不关闭弹窗，点击外部则关闭。
    *   当地图移动时，已打开的弹窗应自动关闭，避免信息与位置不匹配。

### 3.4 权限说明
*   **功能权限**：本模块所有功能，仅对拥有“投研地图”访问权限的用户组开放（如：投研部全员）。该权限由系统后台统一配置。
*   **数据权限**：所有用户查看到的楼盘数据范围一致，为内部数据库全量数据，暂无基于用户角色的行级数据权限控制。

## 4. 非功能性需求

### 4.1 性能需求
*   **页面响应**：开启图层后，地图图标首次渲染时间应小于2秒（在标准网络环境下）。
*   **交互响应**：点击图标弹出详情弹窗的响应时间应小于500毫秒。
*   **搜索响应**：输入关键字后，搜索建议列表的展示延迟应小于300毫秒。
*   **并发量**：系统需支持至少50名投研人员同时在线使用地图图层功能。

### 4.2 兼容性需求
*   **浏览器**：必须兼容 Chrome 90+、Safari 14+、Edge 90+ 的最新两个版本。
*   **设备**：仅限Web端桌面浏览器，暂不要求适配移动端。
*   **屏幕分辨率**：适配主流分辨率（1920*1080， 1440*900）。

### 4.3 安全性需求
*   **接口安全**：所有请求楼盘数据的接口必须进行用户身份鉴权。
*   **数据安全**：前端不存储、不缓存敏感的楼盘业务数据。
*   **防攻击**：对搜索接口实施防SQL注入和XSS攻击处理。

### 4.4 可用性需求
*   **可用性**：系统整体可用性不低于99.5%。
*   **容错性**：当部分数据加载失败时，不影响地图基础功能和其他已加载图层的使用，并给出明确错误提示。
*   **异常提示**：所有用户操作导致的错误（网络、数据、服务器），均需通过友好的Toast或Inline Message提示用户，提示文案需明确。
