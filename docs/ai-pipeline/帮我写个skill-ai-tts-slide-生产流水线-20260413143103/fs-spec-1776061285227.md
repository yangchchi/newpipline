# FS 正文（Markdown）

# FS

## 1. 目标
解决用户从文本快速生成专业级、音画同步的演示视频的效率问题。成功标准：从点击“生成”到可播放，90%的请求在60秒内完成；音画同步误差小于100ms。

## 2. 角色与场景
- **用户**：产品经理、市场专员、咨询顾问等B端内容创作者。
- **使用场景**：需要快速制作产品演示、客户提案、分析报告等汇报材料。

## 3. 功能
### 功能点1：多模式内容输入
- **输入**：
    1.  用户通过顶部Tab选择的模式（脚本/大纲/关键词）。
    2.  用户在对应编辑器输入的文本内容。
- **规则**：
    1.  **模式切换**：If 用户点击模式Tab，then 切换编辑器类型及提示文案。
    2.  **脚本模式**：
        - If 用户输入包含H1标题，then 每个H1及其后续内容（至下一个H1前）被划分为一“页”。
        - If 用户输入包含H2标题，then 该点可能触发页内分节动画。
    3.  **大纲模式**：
        - If 用户输入一级要点（无缩进），then 每个一级要点被转化为一“页”。
        - If 用户输入二级/三级要点（通过Tab缩进），then 该点被转化为页内动画序列元素。
    4.  **提交校验**：If 用户点击“生成演示”时，输入内容字符数 < 10，then 阻止提交并前端提示：“请输入至少10个字符的正文内容”。
- **输出**：统一转化为带层级（页、节）和段落标记的结构化JSON数据。
- **验收标准**：
    1.  切换模式时，编辑器及示例文案即时更新。
    2.  脚本模式下，H1标题能正确分页。
    3.  大纲模式下，缩进层级能正确保留。
    4.  输入少于10字符时，点击生成按钮弹出指定提示。

### 功能点2：风格与音色配置
- **输入**：
    1.  用户点击选择的视觉模板卡片。
    2.  用户选择的显示模式（横屏16:9 / 竖屏9:16）。
    3.  用户选择的AI音色及语速滑块值（0.8x - 1.5x）。
- **规则**：
    1.  **模板选择**：If 用户点击某模板卡片，then 该卡片显示为选中状态，右侧预览图更新为该模板静态效果。
    2.  **显示模式**：If 用户切换横屏/竖屏，then 右侧预览图比例即时变化。
    3.  **音色试听**：If 用户点击“试听”按钮，then 播放一段使用当前选中音色的固定样音。
- **输出**：配置参数集合（template_id, display_mode, voice_id, speed_rate）。
- **验收标准**：
    1.  点击模板、切换显示模式，右侧预览图在500ms内更新。
    2.  点击音色试听，能立即听到对应样音。

### 功能点3：生成演示
- **输入**：功能点1输出的结构化内容数据 + 功能点2输出的配置参数。
- **规则**：
    1.  **触发生成**：If 用户点击“生成演示”按钮且输入校验通过，then 按钮变为加载状态并禁用，向后端发起生成请求。
    2.  **生成过程**：
        - 后端根据内容数据生成视觉页面序列。
        - 后端调用TTS服务，根据内容文本和音色、语速配置生成音频文件。
        - 后端计算时间轴，将视觉动效序列与音频时间点对齐。
    3.  **成功处理**：If 生成成功，then 自动跳转至播放器页面（URL: `/play/:projectId`）。
    4.  **失败处理**：
        - If TTS服务失败，then 提示：“音频生成失败，请稍后重试或尝试更换音色”，并保留已生成的视觉页面（如有）。
        - If 其他生成过程失败，then 提示：“生成失败，请稍后重试”。
- **输出**：
    1.  成功：包含同步时间轴数据的项目数据，用于播放。
    2.  失败：明确的错误提示信息。
- **验收标准**：
    1.  点击生成后，按钮状态正确变化，防止重复提交。
    2.  生成成功后在3秒内跳转到播放页。
    3.  生成失败时，弹出符合规则的错误提示。

### 功能点4：演示播放器
- **输入**：功能点3生成的项目数据（含时间轴）。
- **规则**：
    1.  **自动播放**：If 页面加载完成且数据就绪，then 自动开始播放。
    2.  **播放控制**：
        - If 用户点击“暂停”按钮，then 暂停音频及所有动画。
        - If 用户点击“播放”按钮，then 从当前时间点恢复播放。
    3.  **进度控制**：If 用户拖拽进度条至位置X%，then 计算对应的音频时间点T和动画帧F，并跳转至该位置。
    4.  **控件显隐**：If 用户鼠标在播放区域内移动，then 显示底部控制栏和顶部工具栏；If 用户鼠标5秒无操作，then 隐藏底部控制栏和顶部工具栏。
- **输出**：无。提供用户交互与视听播放。
- **验收标准**：
    1.  播放时，音频与动画同步误差小于100ms。
    2.  拖拽进度条后，音画能准确跳转到对应位置并保持同步。
    3.  鼠标交互能触发控件显隐。

### 功能点5：导出视频
- **输入**：用户在当前播放器页面点击“导出视频”按钮，并选择分辨率（720p/1080p）。
- **规则**：
    1.  **触发导出**：If 用户点击“导出视频”并确认分辨率，then 将渲染任务提交至后台队列，页面显示“渲染中”状态及预计完成时间。
    2.  **成功处理**：If 后台渲染成功，then 通过系统通知告知用户，并提供下载链接。
    3.  **失败处理**：If 后台渲染失败，then 通知用户“视频导出失败，请重新尝试导出”。
    4.  **输出规则**：视频格式为MP4 (H.264编码， AAC音频)，帧率30 fps，文件名：`项目标题_导出日期.mp4`。
- **输出**：可供下载的视频文件（成功）或失败提示。
- **验收标准**：
    1.  提交导出后，页面状态立即更新为“渲染中”。
    2.  渲染成功后，用户能收到通知并下载正确命名的MP4文件。
    3.  渲染失败，用户收到指定失败提示。

## 4. 规则补充
- **全局业务规则1（输入校验）**：If 用户提交的内容经敏感词库匹配命中，then 阻止提交并提示：“内容包含违规词汇，请修改”。
- **全局业务规则2（导出限制-免费用户）**：If 免费用户本月已导出次数 >= 5，then 点击“导出视频”时提示：“本月免费导出次数已用尽，请升级套餐或下月再试”。
- **全局业务规则3（浏览器兼容）**：If 用户使用非Chrome浏览器访问核心功能页，then 页面顶部显示温和提示：“为获得最佳体验，推荐使用最新版Chrome浏览器”。

## 5. 示例（必须）
### 正常：
**Input (功能点1&2&3)**:
- 模式：脚本模式
- 输入内容："# 市场趋势\nAI技术快速发展。\n# 我们的方案\n提供一站式平台。"
- 模板：选择“科技蓝”模板
- 音色：选择“沉稳男声”，语速1.0x
- 操作：点击“生成演示”

**Output**:
- 前端：按钮变为加载状态
- 后端：生成2页视觉内容，合成对应音频，建立同步时间轴
- 结果：约30秒后，自动跳转至 `/play/proj_123`，并开始自动播放音画同步的演示。

**Input (功能点5)**:
- 操作：在播放页面点击“导出视频”，选择“1080p”

**Output**:
- 前端：显示“渲染中，预计需要5分钟...”
- 后端：渲染任务进入队列
- 结果：5分钟后，收到系统通知：“视频渲染完成”，点击通知下载文件 `市场趋势_20240527.mp4`。

### 异常：
**Input (功能点1)**:
- 模式：脚本模式
- 输入内容："你好"
- 操作：点击“生成演示”

**Output**:
- 前端弹出提示框，内容为：“请输入至少10个字符的正文内容”，生成请求未发出。

**Input (功能点3)**:
- （正常输入后）点击“生成演示”，但TTS服务临时不可用。

**Output**:
- 前端弹出提示框，内容为：“音频生成失败，请稍后重试或尝试更换音色”。

**Input (功能点5 - 触发全局规则2)**:
- 用户为免费用户，本月已导出5次视频。
- 操作：点击“导出视频”

**Output**:
- 前端弹出提示框，内容为：“本月免费导出次数已用尽，请升级套餐或下月再试”。
