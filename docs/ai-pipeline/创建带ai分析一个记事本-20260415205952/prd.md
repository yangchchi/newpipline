## 1. 文档概述

### 1.1 文档背景
在当前的团队协作与知识管理场景中，员工普遍使用各类笔记工具记录会议纪要、项目复盘、灵感想法和待办事项。然而，传统的记事本工具仅解决了“记录”问题，未能有效解决“信息消化、整理与再利用”的痛点。具体表现为：
*   **信息过载，提炼困难：** 冗长的会议记录、项目文档需要人工花费大量时间进行总结和要点提取，效率低下。
*   **信息孤岛，关联性弱：** 笔记之间缺乏智能关联，历史笔记中的经验、决策点、待办事项无法被有效追溯和复用。
*   **行动项遗漏，跟进困难：** 从会议纪要中手动识别和分配任务（Action Items）容易遗漏，且缺乏自动跟踪机制。
*   **知识沉淀不足：** 海量的非结构化笔记数据沉淀后，难以被快速检索和深度挖掘其价值。

本项目旨在创建一个“带AI分析的记事本”，通过引入AI能力，将被动记录工具升级为主动的智能知识工作伴侣，解决上述核心痛点。

### 1.2 项目目标
**业务目标（至2026年底）：**
*   上线后6个月内，实现付费企业客户数达到500家。
*   用户平均每周使用AI功能次数 ≥ 3次。
*   将团队会议纪要整理成结构化文档的平均耗时降低50%。

**产品目标：**
*   **MVP阶段（V1.0）：** 实现核心的记事、编辑、基础AI分析（总结、提取、任务识别）、团队共享功能。
*   **V1.5阶段：** 上线智能问答、笔记关联推荐、高级权限管理。
*   **V2.0阶段：** 推出基于团队知识库的定制化AI模型训练、开放API。

**用户价值：**
*   **对普通员工：** 提升个人工作效率，快速从信息中获取洞察，减少重复性整理工作。
*   **对团队管理者：** 确保团队信息同步，清晰跟踪任务进展，强化知识沉淀与传承。
*   **对企业：** 降低信息处理成本，提升组织智商，将非结构化数据转化为可操作的资产。

### 2.1 用户角色
| 角色 | 核心诉求 | 使用场景举例 |
| :--- | :--- | :--- |
| **普通成员** | 快速记录、高效整理、便捷查找历史信息、智能辅助生成内容。 | 记录每日站会、撰写项目周报、整理学习笔记。 |
| **团队管理员** | 管理团队成员、设置笔记权限、监督团队知识沉淀进度、查看团队分析概览。 | 新建项目组工作空间、归档已结束项目的笔记本、查看团队最常使用的AI功能。 |
| **访客/外部协作者** | 仅查看或评论特定笔记，无编辑权限。 | 客户参与项目评审会议，查看并评论会议纪要。 |
| **系统管理员** | 管理整个SaaS平台的用户、计费、监控系统状态。 | 处理企业账户开通、配置系统级参数、查看服务健康度。 |

### 2.2 业务流程
**2.2.1 核心业务流程：创建并利用AI分析笔记**
1.  **用户登录**进入其所属的工作空间。
2.  **创建笔记**：在指定笔记本内，点击“新建笔记”，输入标题和正文。
3.  **触发AI分析**：
    *   **自动触发**：用户保存笔记后，系统自动在后台进行“基础分析”（如：提取关键词、识别潜在任务）。
    *   **手动触发**：用户选中部分文本或在整个笔记页面，点击“AI助手”按钮，选择具体分析功能（如：“生成摘要”、“提炼要点”）。
4.  **呈现分析结果**：分析结果以清晰、可交互的板块形式插入到笔记中（或侧边栏展示）。例如，“智能摘要”板块、“待办事项列表”。
5.  **用户交互**：用户可对AI生成的结果进行：
    *   **接受**：一键将结果插入笔记正文。
    *   **修改**：直接编辑AI生成的内容。
    *   **重新生成**：要求AI基于相同指令重新生成。
    *   **忽略/删除**：移除该分析结果。
6.  **分享与协作**：用户可将笔记链接分享给工作空间内的同事或外部访客，并设置其权限（查看、评论、编辑）。
7.  **后续跟进**：对于识别出的“待办事项”，可被分配给特定成员，并同步至集成的任务管理工具（如Jira, Asana，**此为未来扩展点，V1.0仅标记和展示**）。

**2.2.2 异常流程**
*   **AI服务不可用**：当调用AI服务超时或失败时，界面应友好提示“AI服务暂时不可用，请稍后再试”，并保留手动触发按钮。笔记的创建、保存等基础功能不受影响。
*   **内容违规**：当用户输入或AI生成的内容触发电商安全策略时，系统应拒绝处理并提示“内容不符合使用规范”。
*   **网络中断**：支持离线草稿保存，网络恢复后自动同步。

### 2.3 使用场景
1.  **场景一：会后五分钟产出纪要**
    *   **用户**：项目经理Alice。
    *   **情境**：一小时的产品评审会刚结束，Alice需要立即将凌乱的记录整理成结构化会议纪要，并发出。
    *   **使用流程**：Alice在“产品项目”笔记本中新建笔记，将会议讨论的原始记录粘贴进去。她点击“AI助手” -> “生成会议纪要”，AI自动输出包含“会议主题、参会人、讨论要点、决议、待办事项（明确负责人和截止日期）”的标准格式。Alice稍作修改，一键分享给所有参会者。

2.  **场景二：从长篇报告中快速抓取洞察**
    *   **用户**：市场分析师Bob。
    *   **情境**：Bob收到一份50页的行业分析PDF，他需要快速了解核心结论和竞争对手动态。
    *   **使用流程**：Bob将PDF文本复制到AI记事本中。他使用“提炼核心论点”功能，获得一页纸的摘要。接着，他针对特定章节，使用“智能问答”功能，输入“报告中对X公司未来三年的战略预测是什么？”，AI直接给出基于报告内容的答案。

3.  **场景三：团队项目知识沉淀**
    *   **用户**：技术团队主管Charlie。
    *   **情境**：一个为期半年的项目即将结束，团队产生了大量设计文档、会议记录、踩坑总结。
    *   **使用流程**：Charlie创建一个“XX项目复盘”笔记本，将相关笔记全部移入。他利用“笔记关联”功能，发现系统自动将关于“性能优化”的讨论从不同会议记录中关联了起来。他还可以要求AI“基于本项目所有笔记，生成一份技术架构演进总结”，作为团队知识资产。

## 4. 非功能性需求

### 4.1 性能需求
*   **页面加载**：核心页面（笔记列表、编辑页）在常规网络下首屏加载时间 < 2秒。
*   **AI响应**：常规AI分析请求（如摘要、提炼）的响应时间 95% 在 **3秒** 以内。
*   **并发支持**：系统需支持单工作空间同时在线用户数 ≥ 200人。
*   **数据同步**：笔记保存后，其他协作者在1秒内看到更新提示。

### 4.2 兼容性需求
*   **浏览器**：支持 Chrome 90+, Firefox 88+, Safari 14+, Edge 90+ 的最新两个版本。
*   **客户端**：首期提供Web端，规划移动端App（iOS 15+, Android 11+）。
*   **分辨率**：适配主流桌面分辨率（1920x1080, 1440x900）及平板设备。

### 4.3 安全性需求
*   **数据传输**：全程使用HTTPS (TLS 1.3) 加密。
*   **数据存储**：用户笔记内容在数据库中进行加密存储。
*   **权限控制**：严格遵循RBAC（基于角色的访问控制）模型，前端与后端均需验证。
*   **防攻击**：具备防SQL注入、XSS攻击、CSRF攻击的能力。对AI接口设置频率限制，防止滥用。
*   **隐私合规**：AI服务提供商需符合数据隐私协议，用户笔记内容不用于第三方模型训练。

### 4.4 可用性需求
*   **系统可用性**：SLA不低于99.5%（每月宕机时间不超过约3.6小时）。
*   **容错能力**：AI服务降级时，不影响笔记的基础读写功能。
*   **异常提示**：所有用户操作错误、网络异常、系统错误均需有友好、明确的中文提示，告知用户原因及建议操作。
*   **可访问性**：支持基本的键盘导航和屏幕阅读器识别（符合WCAG 2.1 AA级标准）。
