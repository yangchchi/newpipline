# FS

## 1. 目标
解决员工每日工作记录分散、手工汇总效率低的问题，通过线上化、结构化记录并与项目管理系统打通，实现工时数据自动归集与分析。
**成功标准**：
1.  上线后3个月内，公司90%以上项目成员使用该系统。
2.  财务月度工时统计人力耗时降低70%。
3.  员工平均每日填写耗时低于3分钟。

## 2. 角色与场景
- **用户**：公司项目成员（如工程师、设计师）。
- **使用场景**：员工每日下班前，在Web端或移动端记录当天完成的多项工作任务、工时及所属项目。

## 3. 功能
### 功能点1：添加单条任务记录
- **输入**：
    1.  项目选择（从下拉列表中选择，支持关键词搜索）。
    2.  任务描述（文本）。
    3.  工时（数字）。
    4.  用户点击“添加”按钮。
- **规则**：
    1.  **项目选择规则**：
        - IF 用户未选择项目 THEN 前端提示“请选择项目”，阻止添加。
    2.  **任务描述规则**：
        - IF 输入为空 THEN 前端提示“任务描述不能为空”，阻止添加。
        - IF 输入字符数 > 200 THEN 前端提示“描述不能超过200字”，阻止添加。
    3.  **工时规则**：
        - IF 输入非数字 THEN 前端提示“请输入有效数字”，阻止添加。
        - IF 输入数字 < 0.5 THEN 前端提示“单任务工时不能少于0.5小时”，阻止添加。
        - IF 输入数字 > 12 THEN 前端提示“单任务工时不能超过12小时”，阻止添加。
        - IF 输入数字的小数部分不是 `.0` 或 `.5` THEN 前端提示“工时请输入0.5的倍数（如1, 1.5, 2）”，阻止添加。
    4.  **添加成功规则**：
        - IF 所有输入校验通过 THEN 将任务添加到下方列表，清空输入框，并更新底部“当日总工时”。
- **输出**：
    1.  任务列表新增一条记录（显示项目名、任务描述、工时、删除按钮）。
    2.  底部“当日总工时”数值更新。
- **验收标准**：用户能成功添加一条符合规则的任务到列表中。

### 功能点2：管理任务列表（编辑/删除）
- **输入**：
    1.  用户点击某条任务记录的“删除”图标。
    2.  用户点击某条任务记录进行编辑（双击或点击编辑图标）。
- **规则**：
    1.  **删除规则**：
        - IF 用户点击删除图标 THEN 前端弹出确认框：“确认删除此任务？”
        - IF 用户确认删除 THEN 从任务列表中移除该任务，并更新底部“当日总工时”。
    2.  **编辑规则（提交前）**：
        - IF 用户触发编辑 THEN 该任务行变为可编辑状态（项目、描述、工时变为输入框）。
        - IF 用户修改后保存 THEN 应用【功能点1】的输入规则进行校验。
        - IF 校验通过 THEN 更新该行数据显示，并更新“当日总工时”。
- **输出**：
    1.  任务列表更新（条目删除或内容变更）。
    2.  底部“当日总工时”数值更新。
- **验收标准**：用户能删除或修改已添加的任务，且总工时同步准确更新。

### 功能点3：暂存日报草稿
- **输入**：用户点击“暂存草稿”按钮。
- **规则**：
    1.  IF 任务列表为空 THEN 前端提示“没有可保存的内容”，操作无效。
    2.  IF 任务列表不为空 THEN 将当前所有任务数据（项目ID、描述、工时）保存至本地缓存或后台“草稿”状态。
    3.  IF 保存成功 THEN 前端提示“草稿保存成功”。
    4.  IF 保存失败（如网络错误） THEN 前端提示“保存失败，请检查网络后重试”。
- **输出**：
    1.  系统提示信息。
    2.  数据被保存，用户下次进入页面时自动加载草稿。
- **验收标准**：点击暂存后，当前填写内容被保存，刷新页面后可恢复。

### 功能点4：提交日报
- **输入**：用户点击“提交日报”按钮。
- **规则**：
    1.  **提交前校验规则**：
        - IF 任务列表为空 THEN 前端提示“请至少添加一条任务记录”，阻止提交。
        - IF 任何一条任务未选择项目 THEN 前端提示“请为每项任务关联所属项目”，并高亮未选项目的任务行，阻止提交。
        - IF 底部“当日总工时” <= 0 THEN 前端提示“总工时需大于0”，阻止提交。
        - IF 底部“当日总工时” > 24 THEN 前端提示“总工时不能超过24小时”，阻止提交。
    2.  **提交处理规则**：
        - IF 所有校验通过 THEN 将任务列表数据（用户ID、日期、任务明细）发送至后端。
        - IF 后端保存成功 THEN 前端提示“提交成功”，并将页面按钮置灰/变为“已提交”，任务列表锁定不可编辑。
        - IF 后端保存失败（如服务器错误） THEN 前端提示“提交失败，请稍后重试”。
        - IF 网络中断 THEN 前端提示“网络连接失败，请检查网络后重试”，并建议自动缓存提交数据。
- **输出**：
    1.  系统提示信息（成功/失败）。
    2.  页面状态变更（可编辑 -> 已提交锁定）。
    3.  数据持久化至数据库，并触发与PM系统的同步流程。
- **验收标准**：校验通过后，数据成功提交，页面状态正确变更，用户无法再修改当日记录。

### 功能点5：加载项目列表
- **输入**：用户进入工作记录页面。
- **规则**：
    1.  **加载规则**：
        - 页面加载时，自动从后端请求“有效项目列表”。
        - 后端项目列表数据来自与PM系统的定时同步服务。
    2.  **降级规则**：
        - IF 后端请求失败或超时（>3秒） THEN 使用本地最近一次缓存的列表数据，并显示提示“项目列表加载中，可能非最新”。
    3.  **搜索规则**：
        - IF 用户在项目选择框输入关键词 THEN 实时过滤下拉列表，只显示项目名称包含关键词的项目。
- **输出**：
    1.  项目选择下拉框 populated with 项目数据（项目ID， 项目名称）。
    2.  （可选）加载状态或降级提示。
- **验收标准**：用户能从一个包含所有有效项目的下拉列表中为任务选择项目，支持搜索筛选。

## 4. 规则补充
- **全局业务规则（if-then）**:
    - IF 用户已提交当日报告 THEN 所有“添加”、“编辑”、“删除”、“提交”按钮功能禁用，页面仅可查看。
    - IF 用户有未提交的昨日（或更早）草稿 THEN 进入页面时顶部显示提示：“您有[日期]的未提交草稿，是否继续编辑？”，并提供“查看”入口。
    - IF PM系统同步服务更新了项目列表（如项目关闭）THEN 已关闭项目不再出现在前端下拉选择框中，但不影响历史记录中的显示。

## 5. 示例（必须）
### 正常：
**Input**:
1.  项目选择：`“CRM系统重构”`
2.  任务描述：`“开发用户登录模块后端接口”`
3.  工时：`3.5`
4.  点击“添加”。
5.  再次添加：项目=`“CRM系统重构”`， 描述=`“Code Review”`， 工时=`1`。
6.  点击“提交日报”。

**Output**:
1.  任务列表显示两条记录。
2.  底部显示“当日总工时：4.5小时”。
3.  点击提交后，提示“提交成功”。
4.  页面按钮变为“已提交”，任务列表不可编辑。

### 异常：
**场景1：总工时超限**
**Input**:
1.  添加任务：工时=`10`。
2.  添加任务：工时=`15`。
3.  点击“提交日报”。
**Output**:
前端提示：“总工时不能超过24小时”，提交被阻止。

**场景2：未选项目提交**
**Input**:
1.  添加任务：项目=`空`， 描述=`“开会”`， 工时=`2`。
2.  点击“提交日报”。
**Output**:
前端提示：“请为每项任务关联所属项目”，提交被阻止，且该任务行高亮显示。

**场景3：网络中断后提交**
**Input**:
1.  添加若干有效任务。
2.  断开网络。
3.  点击“提交日报”。
**Output**:
前端提示：“网络连接失败，请检查网络后重试”。（理想情况：提交数据被前端自动缓存，网络恢复后提示用户重新提交）。
