# FS

## 1. 目标
解决员工因AI工具繁多、学习路径不清、缺乏支持而导致的“AI焦虑”问题，通过一站式平台降低AI使用门槛。成功标准：MVP上线后6个月内，平台月活跃用户达到全体员工数量的60%。

## 2. 角色与场景
- **用户**：公司内部员工，分为浏览者/学习者、提问者、需求提交者、AI专家、内容运营者。
- **使用场景**：员工在工作中需要寻找AI工具、学习AI知识、解决AI相关问题或发起AI项目时，使用本平台。

## 3. 用户故事
### 用户故事1：作为浏览者，我希望通过关键词搜索AI工具，以便快速找到符合我需求的工具。
#### 验收标准：
##### （1）用户在顶部全局搜索框或工具广场页面的搜索框中输入关键词并触发搜索。
##### （2）系统在“工具”数据范围内，检索工具名称、简介、适用场景字段中包含该关键词的记录。
##### （3）系统以卡片列表形式返回所有匹配的记录，并显示工具名称、简介、标签。
##### （4）如果未找到任何匹配记录，系统在结果区域显示提示信息：“未找到相关工具，您可以尝试调整关键词，或前往「需求广场」提交需求”。

### 用户故事2：作为提问者，我希望向AI提问并获得基于公司内部知识的答案，以便解决工作中的具体问题。
#### 验收标准：
##### （1）用户进入“智能问答”页面，在输入框中输入自然语言问题并发送。
##### （2）系统调用RAG模型，基于已上传的知识库文档生成答案。
##### （3）答案生成期间，输入框旁显示加载动画。
##### （4）答案生成后，系统在对话区域以气泡形式展示答案文本。
##### （5）答案末尾需注明参考来源，格式为：“参考来源：《文档名称》”。
##### （6）如果知识库中未检索到相关问题信息，系统返回答案：“知识库中暂无此问题答案，您可以将此问题转为「需求」提交，或通过反馈入口联系我们。”

### 用户故事3：作为需求提交者，我希望提交一个AI项目需求，以便获得公司内部专家的技术支持。
#### 验收标准：
##### （1）用户（已登录）在“需求广场”点击“提交需求”按钮。
##### （2）系统跳转至需求提交表单页，表单包含字段：需求标题（必填）、详细描述（必填）、业务价值、期望完成时间、联系人邮箱（必填）。
##### （3）用户填写必填字段后，点击“提交”按钮。
##### （4）系统将需求状态初始化为“待审核”，并通知内容运营人员。
##### （5）内容运营人员在后台审核通过后，需求状态变为“待认领”，并公开显示在需求广场列表页。
##### （6）如果用户未填写任一必填字段，则“提交”按钮为禁用状态；提交时前端校验必填字段，若为空则提示：“请填写[字段名]”。

### 用户故事4：作为AI专家，我希望认领一个公开的AI需求，以便对接需求并开展项目。
#### 验收标准：
##### （1）用户（AI专家角色）在“需求广场”浏览状态为“待认领”的需求列表。
##### （2）用户点击某个需求卡片，进入需求详情页。
##### （3）在详情页，用户点击“认领此需求”按钮。
##### （4）系统将需求状态从“待认领”更新为“进行中”，并在需求详情页公开显示认领者的姓名或ID。
##### （5）系统通过站内消息通知需求的原提交者：“您的需求「[需求标题]」已被[专家姓名]认领”。
##### （6）如果需求状态不是“待认领”（如“进行中”、“已完结”），“认领此需求”按钮不显示。

### 用户故事5：作为内容运营者，我希望上传新的知识库文档，以更新智能问答的知识范围。
#### 验收标准：
##### （1）用户（内容运营者角色）登录后台管理系统，进入“知识库管理”页面。
##### （2）用户点击“上传文档”按钮，选择本地文件（支持.pdf， .docx， .txt格式）。
##### （3）如果文件大小超过50MB，系统前端立即提示：“文件大小超过50MB限制，请重新选择”。
##### （4）如果文件格式不符合要求，系统前端立即提示：“仅支持.pdf， .docx， .txt格式文件”。
##### （5）文件通过校验后，系统开始异步上传和处理，并在文档列表中新增一条记录，状态显示为“处理中”。
##### （6）文档处理完成后，系统更新该记录状态为“可用”。
##### （7）如果处理失败，系统更新该记录状态为“失败”，并可在后台查看失败原因。

## 4. 规则补充
### 全局业务规则
1.  **身份验证规则**：
    *   IF 用户尝试执行“提交需求”或“提交最佳实践”操作。
    *   THEN 系统检查用户登录状态，如果未登录，则跳转至登录页。
2.  **权限控制规则**：
    *   IF 用户尝试访问后台管理功能（如知识库管理、内容审核）。
    *   THEN 系统校验用户角色是否为“内容运营者”或“系统管理员”，如果不是，则返回“403无权限”错误。
3.  **需求超时规则**：
    *   IF 一个需求状态为“待认领”且创建时间超过7天。
    *   THEN 系统自动发送邮件通知预设的运营管理员。
4.  **搜索范围规则**：
    *   IF 用户在首页顶部全局搜索框进行搜索。
    *   THEN 系统同时在“工具”、“最佳实践案例”、“学习教程”三个数据范围内进行检索，并分类展示结果。
    *   IF 用户在“工具广场”页面的搜索框进行搜索。
    *   THEN 系统仅在“工具”数据范围内进行检索。
5.  **内容可见性规则**：
    *   IF 内容（工具、案例、教程）被运营人员在后台标记为“下线”。
    *   THEN 该内容不再出现在任何用户端列表和搜索结果中，但通过直接URL访问详情页时显示“该内容已下线”。
