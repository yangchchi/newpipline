# FS 正文（Markdown）

# FS

## 1. 目标
为管理层提供自动化、可视化的数据看板，实时监控工作日志填写情况，提升管理效率与决策依据。
**成功标准**：
1.  管理员查看全公司日志整体情况的时间从平均30分钟/次缩短至1分钟以内。
2.  全公司日志提交率从85%提升至95%。

## 2. 角色与场景
- **用户**：部门负责人、系统管理员/公司高管。
- **使用场景**：
    1.  部门负责人每日晨会前快速检查团队昨日日志提交情况。
    2.  管理员每月初查看上月全公司及各部门的日志填写趋势与对比。

## 3. 功能
### 功能点1：全局筛选器
- **输入**：
    1.  维度选择（单选）：部门 / 员工。
    2.  时间粒度（单选）：日 / 周 / 月。
    3.  时间范围选择器：根据时间粒度显示为日期选择器、周选择器或月选择器。
    4.  查询按钮。
    5.  重置按钮。
- **规则**：
    1.  IF 用户选择“部门”维度 THEN 对比柱状图X轴显示部门名称。
    2.  IF 用户选择“员工”维度 THEN 对比柱状图X轴显示员工姓名（默认展示提交率最低的Top 20）。
    3.  IF 用户切换“时间粒度” THEN 时间范围选择器组件类型同步切换。
    4.  IF 用户点击“查询”或“时间范围”变更 THEN 触发看板数据重新加载。
    5.  IF 用户点击“重置” THEN 所有筛选条件恢复为默认值（维度=部门，时间粒度=日，时间范围=昨日）。
- **输出**：
    1.  向后台API传递筛选参数 `{dimension: ‘dept’/‘user’, granularity: ‘day’/‘week’/‘month’, startDate: ‘YYYY-MM-DD’, endDate:‘YYYY-MM-DD’}`。
    2.  触发看板数据刷新。
- **验收标准**：
    1.  页面加载后，筛选器默认值正确。
    2.  切换时间粒度，时间选择器组件类型正确联动变化。
    3.  点击查询，看板数据更新为对应筛选结果。

### 功能点2：核心指标卡 (KPI Cards)
- **输入**：来自功能点1的筛选参数。
- **规则**：
    1.  **日志提交率** = (已提交日志数 / 应提交日志总数) * 100%。
        - IF 应提交总数 > 0 THEN 计算并展示百分比（保留一位小数）。
        - IF 应提交总数 = 0 THEN 显示“--”。
    2.  **日志及时率** = (及时提交日志数 / 已提交日志总数) * 100%。
        - IF 已提交总数 > 0 THEN 计算并展示百分比（保留一位小数）。
        - IF 已提交总数 = 0 THEN 显示“--”。
    3.  **平均填写时长** = Σ(单条日志填写时长) / 已提交日志总数。
        - IF 已提交总数 > 0 THEN 计算结果（单位：分钟），取整展示。
        - IF 计算结果 >= 60分钟 THEN 格式化为“X小时Y分钟”。
        - IF 已提交总数 = 0 THEN 显示“--”。
- **输出**：
    1.  三个独立的指标卡组件，分别展示提交率、及时率、平均填写时长的计算结果。
- **验收标准**：
    1.  指标卡数值随筛选条件变化实时更新。
    2.  数据为0或空时，按规则显示“--”。

### 功能点3：提交情况趋势图
- **输入**：来自功能点1的筛选参数。
- **规则**：
    1.  图表类型：双线折线图。
    2.  X轴：时间。根据筛选的“时间粒度”显示具体日期、周次或月份标签。
    3.  Y轴：比率（0%-100%）。
    4.  数据线：两条。线1为“提交率”趋势，线2为“及时率”趋势。
    5.  IF 查询时间范围内无数据 THEN 图表区显示“暂无数据”占位图。
    6.  IF 用户鼠标悬停在数据点上 THEN 显示Tooltip，内容为 `{时间点: 提交率: X%, 及时率: Y%}`。
- **输出**：一个渲染完成的折线图。
- **验收标准**：
    1.  图表根据筛选条件正确绘制两条趋势线。
    2.  悬停交互正常，Tooltip信息准确。

### 功能点4：部门/员工对比柱状图
- **输入**：来自功能点1的筛选参数。
- **规则**：
    1.  IF 筛选维度为“部门” THEN：
        - 图表类型：分组柱状图。
        - X轴：部门名称（用户有权限查看的部门）。
        - Y轴：日志数量。
        - 数据系列：每个部门对应三根柱子，分别为“应提交数”、“已提交数”、“及时提交数”。
    2.  IF 筛选维度为“员工” THEN：
        - 图表类型：分组柱状图。
        - X轴：员工姓名（默认按“提交率”从低到高排序，取Top 20）。
        - Y轴：比率（0%-100%）。
        - 数据系列：每个员工对应两根柱子，分别为“提交率”、“及时率”。
    3.  IF 查询结果无数据 THEN 图表区显示“暂无数据”占位图。
    4.  IF 用户鼠标悬停在柱子上 THEN 显示Tooltip，内容为对应维度的具体数值。
- **输出**：一个渲染完成的分组柱状图。
- **验收标准**：
    1.  切换“维度”筛选，图表类型和数据系列正确切换。
    2.  员工维度下，默认排序正确（提交率升序）。

## 4. 规则补充
- **全局业务规则（if-then）**：
    1.  **数据权限**：
        - IF 用户角色是“部门负责人” THEN 所有数据查询自动附加其管辖部门ID作为过滤条件。
        - IF 用户角色是“系统管理员” THEN 数据查询无部门限制。
        - IF 用户无访问权限 THEN 拦截请求，看板区域显示“暂无查看权限”。
    2.  **数据时效**：
        - IF 统计数据的最后更新时间为T-1日23:59:59 THEN 看板页面显示“数据更新至 [YYYY-MM-DD]”。
        - IF 数据更新任务失败，当前数据已过期（超过24小时）THEN 看板显示“数据更新延迟，最后更新于 [时间]”。
    3.  **默认视图**：
        - IF 用户首次进入看板页面或点击“重置” THEN 自动以 `{维度=‘部门’， 时间粒度=‘日’， 时间范围=‘昨日’}` 为条件查询并渲染数据。

## 5. 示例（必须）
### 正常：
**Input**:
- 用户：研发部张经理（部门负责人）
- 筛选条件：维度=部门，时间粒度=日，时间范围=2026-04-15
- 后台数据：张经理管辖的研发部，2026-04-15日应提交日志50条，已提交48条，其中及时提交45条。总填写时长为1200分钟。

**Output**:
- 指标卡：提交率: 96.0%， 及时率: 93.8%， 平均填写时长: 25分钟。
- 趋势图：显示2026-04-15单点的提交率(96.0%)和及时率(93.8%)。
- 对比柱状图：X轴仅显示“研发部”，三根柱子分别对应50(应提交)，48(已提交)，45(及时提交)。

### 异常：
**Input**:
- 用户：研发部张经理（部门负责人）
- 筛选条件：维度=员工，时间粒度=周，时间范围=2026年第16周（该周团队无人填写日志）
- 后台数据：查询结果为空。

**Output**:
- 指标卡：提交率: --， 及时率: --， 平均填写时长: --。
- 趋势图：显示“暂无数据”占位图。
- 对比柱状图：显示“暂无数据”占位图。
