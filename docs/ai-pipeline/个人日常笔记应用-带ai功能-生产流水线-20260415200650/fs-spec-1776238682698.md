# FS 正文（Markdown）

# FS

## 1. 目标
解决个人日常记录效率低、信息检索难、价值未挖掘的问题，通过AI辅助实现高效记录、智能整理与语义检索。成功标准：AI功能（摘要/分类）准确率 > 90%；核心笔记创建到AI整理完成平均时长 < 2分钟。

## 2. 角色与场景
- 用户：学生、职场人士、知识工作者/创作者。
- 使用场景：课堂/会议记录、灵感收集、知识整理与复习。

## 3. 功能
### 功能点1：多格式笔记创建
- 输入：
    1.  用户操作：点击“新建笔记”按钮。
    2.  内容数据：文本字符、图片文件（≤10MB）、语音流（单次≤30分钟）。
- 规则：
    1.  IF 用户选择文本输入 THEN 展示文本编辑器。
    2.  IF 用户选择图片输入 THEN 调用系统相册/相机，用户选择/拍摄后，图片插入编辑器。
    3.  IF 用户选择语音输入 THEN 开始录音，录音流实时显示，最长30分钟后自动停止或用户手动停止。
    4.  IF 用户点击“保存”按钮 THEN 触发规则5。
    5.  IF 笔记内容包含图片 THEN 后台异步启动OCR识别，将识别文本附加到笔记内容中。
    6.  IF 笔记内容包含语音 THEN 后台异步启动ASR转写，将转写文本附加到笔记内容中。
    7.  保存后，笔记进入“待AI处理”状态。
- 输出：
    1.  界面：笔记保存成功提示。
    2.  数据：一条包含`note_id`、`content`(纯文本)、`create_time`、`status`=`pending_ai`的笔记记录。
- 验收标准：
    1.  用户可在2秒内完成从点击“新建”到开始输入的流程。
    2.  图片超10MB或语音超30分钟时，保存前明确提示限制。
    3.  保存后，笔记在列表中可见。

### 功能点2：AI自动摘要与关键词生成
- 输入：
    1.  触发器：笔记保存后，且其纯文本内容长度 ≥ 50字符。
    2.  数据：笔记的纯文本内容（原生文本+OCR结果+ASR结果）。
- 规则：
    1.  IF 用户设置AI处理模式为“云端”且网络正常 THEN 调用云端AI服务。
    2.  IF 用户设置AI处理模式为“本地” OR (模式为“云端”但网络异常) THEN 调用本地轻量AI模型。
    3.  IF AI服务调用成功 THEN 解析返回的JSON，提取`summary`和`keywords`。
    4.  IF AI服务调用失败（超时或返回错误）THEN 将笔记`status`置为`ai_failed`，并记录错误日志。
- 输出：
    1.  数据：更新笔记记录，添加`summary`(字符串，长度为原文的20%-30%)、`keywords`(字符串数组，长度3-5)、`status`=`processed`或`ai_failed`。
    2.  界面：在笔记详情页展示AI生成结果区域。
- 验收标准：
    1.  本地模型处理500字内文本，响应时间 < 3秒。
    2.  云端模型（网络良好）端到端响应时间 < 5秒。
    3.  生成的摘要和关键词，经抽样评估准确率 > 90%。

### 功能点3：AI智能分类建议
- 输入：
    1.  同功能点2的纯文本内容。
    2.  系统预置通用标签库。
    3.  用户历史使用过的标签列表。
- 规则：
    1.  AI处理时，连同摘要、关键词一并请求分类建议。
    2.  IF AI返回建议标签 THEN 从返回的标签数组中，筛选出最多3个标签。
    3.  筛选优先级：1) 与用户历史标签匹配的；2) 预置通用标签库中的。
- 输出：
    1.  数据：更新笔记记录，添加`suggested_tags`(字符串数组，长度1-3)。
    2.  界面：在笔记详情页的AI生成区或标签编辑区，以“气泡”形式展示建议标签，并后缀“AI”标识。
- 验收标准：
    1.  用户历史标签被优先建议。
    2.  建议标签与笔记内容主题相关。

### 功能点4：语义搜索
- 输入：
    1.  用户在搜索框输入的查询字符串。
- 规则：
    1.  IF 用户输入查询字符串 THEN 同时执行以下搜索：
        a.  关键词匹配：在笔记的`title`、`keywords`、`content`字段中进行匹配。
        b.  语义匹配：使用本地轻量嵌入模型，将查询和所有笔记的`content`向量化，计算余弦相似度。
    2.  对每条笔记，计算综合得分：`综合得分 = 关键词匹配得分 * 0.4 + 语义相似度得分 * 0.6`。
    3.  按综合得分降序排序，返回前50条结果。
    4.  IF 无任何匹配结果 THEN 返回空列表，并触发友好提示。
- 输出：
    1.  界面：展示搜索结果列表，每条结果包含标题（内容首行截取）、摘要预览（前50字）、标签、时间。
    2.  性能：在5000条笔记量级下，结果显示延迟 < 1秒。
- 验收标准：
    1.  支持自然语言查询（如“上周关于产品的讨论”）。
    2.  搜索结果排序符合相关性预期。

### 功能点5：用户对AI结果的交互（采纳/修改/忽略）
- 输入：
    1.  用户对AI生成结果区域的操作（点击“采纳”、“编辑”或“忽略”）。
- 规则：
    1.  IF 用户点击“采纳” THEN 将`summary`、`keywords`、`suggested_tags`正式关联到笔记，并移除“AI”标识。
    2.  IF 用户点击“编辑” THEN 允许用户修改`summary`、`keywords`、`tags`，修改后视为用户手动添加。
    3.  IF 用户点击“忽略” THEN 关闭AI生成结果区域，本次AI生成结果不关联到笔记。
- 输出：
    1.  数据：更新笔记的`summary`、`keywords`、`tags`字段。
    2.  界面：AI生成结果区域根据用户操作更新或消失。
- 验收标准：
    1.  用户操作后，笔记的元数据立即更新。
    2.  忽略操作后，AI区域可再次通过手动触发显示。

## 4. 规则补充
- 全局业务规则（if-then）
    1.  IF 用户未登录 THEN 所有AI处理使用本地模型；数据仅存储于本地。
    2.  IF 用户已登录且开启云同步 THEN 笔记数据在加密后同步至云端。
    3.  IF 设备存储空间不足 THEN 阻止新笔记创建，并提示明确错误信息：“设备存储空间不足，请清理后重试”。
    4.  IF 云端AI处理请求网络超时（>10秒）THEN 自动降级为使用本地模型处理，并提示：“网络超时，已使用本地AI处理”。
    5.  IF 笔记纯文本内容长度 < 50字符 THEN 跳过AI自动处理（摘要、关键词、分类建议）。

## 5. 示例（必须）
### 正常：
**场景：保存会议记录文本，触发AI处理**
Input:
- 用户操作：在编辑器中输入文本“本周项目会：1. 确定V1.2版本需求。2. 前端延期，需协调资源。3. 下周二进行设计评审。”，并点击保存。
- 用户设置：AI处理模式为“云端”。
- 网络状态：良好。

Output:
- 数据：笔记记录更新，包含：
    - `summary`: “会议确定了V1.2版本需求，指出前端延期需要协调资源，并安排了下周二的设计评审。”
    - `keywords`: [“项目会”, “V1.2”, “前端延期”, “设计评审”]
    - `suggested_tags`: [“会议”, “项目A”, “周报”]
    - `status`: `processed`
- 界面：笔记详情页展示AI生成区域，包含上述摘要、关键词和建议标签。

**场景：语音记录并搜索**
Input:
- 用户操作：录制一段语音“今天读了一篇关于大模型推理优化的文章，提到KV缓存和量化的技术”。
- 后续操作：在搜索框输入“如何优化大模型速度”。
Output:
- 搜索结果列表中包含上述语音转写的笔记，因为语义相似。

### 异常：
**场景：网络异常下的AI处理**
Input:
- 用户操作：保存一篇长文本笔记。
- 用户设置：AI处理模式为“云端”。
- 网络状态：断开。

Output:
- 界面：弹出提示框：“网络不佳，AI处理延迟。选项：1. 仅保存笔记，稍后重试AI处理；2. 使用本地模型处理（功能可能受限）”。
- IF 用户选择选项1 THEN 笔记`status`=`pending_ai`，待网络恢复后重试。
- IF 用户选择选项2 THEN 调用本地模型处理，后续同正常流程。

**场景：AI服务完全失败**
Input:
- 用户操作：保存笔记。
- AI服务状态：云端和本地模型均返回内部错误。

Output:
- 数据：笔记`status`=`ai_failed`。
- 界面：在笔记详情页显示提示：“AI服务暂时不可用，已保存原始笔记。您可以[手动编辑摘要和标签]。”，其中“手动编辑摘要和标签”为可点击入口。

**场景：搜索无结果**
Input:
- 用户在搜索框输入：“我去年夏天去海南旅游的攻略”。
- 系统：无任何笔记包含相关关键词或语义匹配。

Output:
- 界面：展示提示信息：“未找到相关笔记，尝试更换关键词或使用更口语化的描述？”，下方推荐显示最近创建的3条笔记。
