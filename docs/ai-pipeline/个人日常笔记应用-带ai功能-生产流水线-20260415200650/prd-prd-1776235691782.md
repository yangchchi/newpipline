## 1. 文档概述

### 1.1 文档背景
在信息爆炸的时代，个人知识管理与日常记录的需求日益增长。传统笔记工具存在以下痛点：
1.  **记录效率低**：纯手动输入、整理、归档耗时耗力，尤其是在处理长内容或语音时。
2.  **信息检索难**：随着笔记数量增多，基于标题或简单关键词的搜索难以快速定位所需信息。
3.  **信息价值未挖掘**：记录的内容停留在“存储”层面，缺乏有效的二次加工（如摘要、关联、洞察），难以转化为个人知识资产。
4.  **隐私安全顾虑**：个人思考、学习笔记、工作记录等敏感数据，用户对云端存储的隐私性存在担忧。

本项目旨在开发一款面向个人的智能笔记应用，通过深度集成AI能力，解决上述痛点，实现从“记录”到“理解与洞察”的跃迁，成为用户得力的“第二大脑”。

### 1.2 项目目标
#### 业务目标 (上线后6个月内)
1.  **用户增长**：实现注册用户数达到50万，月活跃用户（MAU）达到15万。
2.  **用户留存**：次月用户留存率不低于40%。
3.  **核心功能渗透**：超过60%的活跃用户使用过AI摘要或智能分类功能。

#### 产品目标
1.  **核心流程体验**：用户从创建笔记到完成AI整理（摘要/分类）的平均时长不超过2分钟。
2.  **AI功能准确率**：AI自动摘要、关键词提取、分类的准确率（基于用户反馈及抽样评估）> 90%。
3.  **系统可靠性**：核心功能（创建、保存、搜索）可用性达到99.9%。

#### 用户价值
1.  **提升记录效率**：支持多格式快速录入，语音转文字，降低记录门槛。
2.  **提升整理效率**：AI自动完成摘要、打标签、分类，让整理工作“一键完成”。
3.  **提升检索与回顾效率**：基于语义的全文搜索，让用户能用自然语言快速找到模糊记忆中的内容。
4.  **保障隐私安全**：本地优先的数据存储策略，云端同步可选且端到端加密，让用户安心记录。

## 2. 业务说明

### 2.1 用户角色
| 角色 | 核心特征与诉求 | 使用场景举例 |
| :--- | :--- | :--- |
| **学生** | 需要记录课堂笔记、整理学习资料、备考复习。诉求是快速捕捉、高效归纳、便于检索回顾。 | 上课时录音并转文字，课后一键生成要点摘要。 |
| **职场人士** | 记录会议纪要、项目灵感、待办事项、工作总结。诉求是提升工作效率，信息不遗漏，快速分享。 | 开会时快速记录要点，AI自动提炼行动项和结论。 |
| **知识工作者/创作者** | 收集行业信息、记录灵感瞬间、撰写文章草稿、构建个人知识体系。诉求是深度思考辅助、知识关联、激发创意。 | 阅读长文时保存精华，AI辅助生成灵感提纲或关联过往笔记。 |

### 2.2 业务流程
#### 2.2.1 核心业务流程：创建与智能整理笔记
**正常流程：**
1.  **启动记录**：用户打开App，点击“新建笔记”按钮。
2.  **选择输入方式**：用户可选择纯文本输入、拍照/上传图片、或录制语音。
3.  **内容录入**：
    -   **文本**：直接输入或粘贴。
    -   **图片**：拍摄或从相册选择，系统后台进行OCR文字识别（可选，需用户触发）。
    -   **语音**：录制语音，系统实时或完成后进行ASR转文字。
4.  **保存与AI处理**：用户点击“保存”。系统将笔记内容（文本）发送至AI服务（根据用户设置，可选本地轻量模型或云端高性能模型）。
5.  **AI生成结果**：AI服务异步返回处理结果，包括：
    -   生成一篇简洁的摘要（约占原文长度的20%-30%）。
    -   提取3-5个核心关键词。
    -   建议1-3个分类标签（基于系统预置分类体系或个人历史标签）。
6.  **结果展示与确认**：应用界面展示AI生成结果。用户可进行“采纳”、“修改”或“忽略”操作。若采纳，结果自动关联到该笔记。
7.  **完成存储**：笔记及其元数据（标题、时间、AI结果、标签等）被加密存储于本地数据库。若用户开启云同步，则加密后同步至云端。

**异常流程：**
-   **网络异常（云端AI处理时）**：提示用户“网络不佳，AI处理延迟”，并询问是否“仅保存笔记，稍后重试AI处理”或“使用本地模型处理（功能可能受限）”。
-   **AI处理失败**：提示用户“AI服务暂时不可用，已保存原始笔记”，并提供手动编辑摘要/标签的入口。
-   **存储空间不足**：明确提示用户设备存储空间不足，并引导用户清理缓存或删除不必要笔记。

#### 2.2.2 核心业务流程：检索与回顾笔记
**正常流程：**
1.  **发起搜索**：用户在搜索框输入查询词，可以是关键词（如“会议”）、自然语言问题（如“上周关于产品规划的讨论要点”）或日期（如“2024-05-10”）。
2.  **执行搜索**：系统同时进行：
    -   **关键词匹配**：在标题、标签、正文中匹配。
    -   **语义匹配**：通过嵌入模型将查询和笔记内容向量化，计算相似度。
3.  **结果排序与展示**：综合关键词相关度、语义相似度、笔记时间等因素进行排序，以列表形式展示（包含标题、摘要预览、时间、标签）。
4.  **查看详情**：用户点击搜索结果，进入笔记详情页进行查看或编辑。

**异常流程：**
-   **无搜索结果**：展示友好提示：“未找到相关笔记，尝试更换关键词或使用更口语化的描述？”，并推荐最近创建的笔记。

### 2.3 使用场景
**场景一：学生的课堂笔记高效整理**
> 小王在听一场关于“机器学习”的线上讲座。他打开“智记”，点击录音按钮。讲座结束后，他保存录音。App自动将1小时的音频转为文字稿，并生成一份约200字的讲座摘要，提取了“监督学习”、“神经网络”、“过拟合”等关键词，并建议归类到“学习/AI”标签下。小王复习时，只需阅读摘要，并可通过搜索“过拟合的解决方法”快速定位到原文相关段落。

**场景二：职场人士的会议纪要自动化**
> 小李在参加项目周会。她使用“智记”快速记录同事发言的重点文本。会议结束时，她点击保存。AI自动将零散的记录点整理成结构化的纪要，包括“讨论议题”、“达成共识”、“待办事项（@责任人）”，并建议了“项目A”、“周会”等标签。小李稍作修改即可分享给团队。

**场景三：知识工作者的灵感管理与关联**
> 设计师老张在浏览设计网站时获得灵感，他截图并粘贴到“智记”中，简单附注“新拟态风格配色参考”。保存后，AI不仅生成了图片中主要颜色的色码摘要，还根据“新拟态”、“配色”等关键词，在笔记历史中找到了他半年前记录的关于“设计趋势”的笔记，并提示“可能与您之前的记录相关”。这帮助老张建立了跨时间的知识连接。

## 4. 非功能性需求

### 4.1 性能需求
-   **页面响应**：列表页加载时间（冷启动）< 2秒，笔记详情页打开 < 1秒。
-   **AI处理延迟**：
    -   本地轻量模型处理（摘要/关键词）：平均响应时间 < 3秒（针对500字内文本）。
    -   云端模型处理（网络良好时）：平均端到端响应时间 < 5秒。
-   **搜索性能**：输入查询后，结果显示延迟 < 1秒（在5000条笔记量级下）。
-   **数据同步**：增量笔记同步到云端应在后台静默完成，网络良好时延迟 < 30秒。

### 4.2 兼容性需求
-   **移动端**：
    -   iOS: 支持 iOS 14 及以上版本。
    -   Android: 支持 Android 10 (API Level 29) 及以上版本，并适配主流厂商（华为、小米、OPPO、Vivo等）的常见机型。
-   **Web端**：支持 Chrome 90+, Firefox 85+, Safari 14+ 及相应内核的Edge等现代浏览器。
-   **屏幕适配**：移动端需适配不同屏幕尺寸和分辨率，支持横竖屏切换（笔记编辑页建议锁定竖屏）。

### 4.3 安全性需求
1.  **数据加密**：
    -   本地数据库必须加密（使用SQLCipher或系统提供的安全存储API）。
    -   云端传输使用HTTPS TLS 1.2+。
    -   **【约束】** 云端同步的数据必须采用端到端加密（E2EE），用户密码/密钥本地派生，服务器不存储明文笔记内容。
2.  **权限控制**：应用仅申请必要的设备权限（如麦克风用于录音、相册/相机用于图片、网络用于同步）。
3.  **隐私合规**：
    -   明确用户隐私政策，告知数据如何被收集和使用（尤其是AI处理环节）。
    -   AI处理若使用云端服务，需提供明确的选项，并确保用户数据不用于模型训练（除非用户明确同意）。
    -   遵循GDPR、CCPA等主要市场的数据保护法规精神。

### 4.4 可用性需求
-   **系统可用性**：核心笔记创建、保存、本地检索功能，要求99.9%可用（排除设备本身问题）。
-   **离线可用**：核心的笔记创建、编辑、查看、本地搜索、本地AI处理（若模型已下载）必须支持完全离线操作。
-   **容错与提示**：
    -   网络操作失败应有明确、友好的提示，并提供重试或离线继续的选项。
    -   应用崩溃率（通过监控平台统计）需低于0.1%。
    -   提供数据导出功能（如Markdown、TXT格式），降低用户迁移成本。
