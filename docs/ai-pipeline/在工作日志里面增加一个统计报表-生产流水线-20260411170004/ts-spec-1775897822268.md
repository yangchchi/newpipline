## 4. 核心流程
### 4.1 查询统计流程
```
1. 接收查询请求
2. 参数校验:
   - dimension ∈ ['department', 'employee']
   - targetIds 非空
   - startDate ≤ endDate
   - endDate - startDate ≤ 365天
3. 权限校验:
   - IF user.role = 'department_manager':
       验证 targetIds ⊆ user.managedDepartments
   - ELSE IF user.role ∈ ['hr', 'admin']:
       允许访问全公司
   - ELSE:
       返回403
4. 获取有效员工列表:
   - WHERE status = 'active'
   - AND employmentType = 'formal'
   - AND (startDate ≤ 入职日期 ≤ endDate OR 入职日期 < startDate)
5. 计算指标:
   - fillRate = (COUNT(DISTINCT WorkLog.userId) / totalEmployees) * 100
   - onTimeRate = (COUNT(DISTINCT CASE WHEN isOnTime THEN WorkLog.userId END) / totalEmployees) * 100
   - avgDuration = SUM(submitTime - startEditTime) / COUNT(WorkLog.id)
6. 构建明细数据:
   - IF dimension = 'department':
       按部门分组统计
   - ELSE:
       按员工分组统计
7. 排序分页:
   - ORDER BY sortField sortOrder
   - LIMIT pageSize OFFSET (page-1)*pageSize
8. 返回结果
```

### 4.2 定时任务流程
```
1. 每日02:00触发
2. FOR 过去365天内每个日期区间:
   - 查询所有在职正式员工
   - 计算各维度统计指标
   - 写入StatisticsCache
3. 清理超过365天的缓存
```

### 4.3 导出流程
```
1. 调用查询流程获取完整数据
2. 构建Excel:
   - 创建工作簿
   - 根据dimension设置表头
   - 填充数据
   - 应用格式
3. 生成文件名:
   - `日志填写统计_${dimension}_${targetName}_${startDate}-${endDate}_${timestamp}.xlsx`
4. 返回文件流
```

## 5. 异常处理
- 错误规则：
  - 参数缺失或格式错误 → 返回400
  - 无访问权限 → 返回403
  - 数据库查询失败 → 返回500
  - 导出文件生成失败 → 返回500
  - 缓存数据不存在 → 实时计算
