# FS

## 1. 目标
为管理者提供工作日志填写情况的量化统计视图，以提升管理效率和数据驱动的决策能力。
**成功标准**：
1.  管理者查看团队日志填写情况的平均耗时从人均每日15分钟降低至2分钟以内。
2.  全公司正式员工月度日志整体填写率从85%提升至95%以上。

## 2. 角色与场景
- **用户**：部门负责人、人力资源专员、系统管理员、公司高管。
- **使用场景**：
    1.  **日常检查**：部门负责人每日快速查看本部门成员昨日日志填写情况。
    2.  **周期复盘**：HR每月统计全公司各部门日志按时提交率，用于月度报告。
    3.  **问题排查**：管理者通过平均填写时长等指标，评估日志填写质量。

## 3. 功能
### 功能点1：多维度数据查询
- **输入**：
    1.  统计维度（单选：“部门”或“员工”）。
    2.  目标对象（维度为“部门”时：单选一个部门；维度为“员工”时：可多选员工）。
    3.  统计周期（选择预置周期或自定义起止日期）。
- **规则**：
    1.  **部门选择规则 (R1.1)**：
        - IF 用户角色是“部门负责人” THEN 部门选择器仅展示其管辖的部门树。
        - IF 用户角色是“人力资源专员”或“系统管理员” THEN 部门选择器展示全公司部门树。
    2.  **员工选择规则 (R1.2)**：
        - IF 维度为“员工” THEN 员工选择器默认列出用户所属（或有权限查看）部门下的员工，支持按姓名/工号搜索。
    3.  **周期选择规则 (R1.3)**：
        - IF 选择自定义日期范围 THEN 结束日期 - 开始日期 ≤ 365 天。
    4.  **查询触发规则 (R1.4)**：
        - IF 用户点击“查询”按钮 THEN 系统根据当前输入条件，刷新下方数据概览区与明细区。
        - IF 用户点击“重置”按钮 THEN 清空所有输入条件至默认状态（维度：部门；部门：用户所属第一个部门；周期：本周），不触发数据刷新。
- **输出**：
    1.  查询条件被提交。
    2.  触发功能点2和功能点3的数据刷新。
- **验收标准**：
    1.  用户可成功设置并提交查询条件。
    2.  权限控制生效，用户无法看到无权访问的部门或员工。

### 功能点2：核心指标数据展示
- **输入**：来自功能点1的查询条件。
- **规则**：
    1.  **数据范围规则 (R2.1)**：所有统计仅针对查询周期内处于“在职”状态的“正式员工”。
    2.  **员工填写率计算规则 (R2.2)**：
        - `填写率 = (实际填写日志的员工数 / 应填写员工总数) * 100%`
        - `应填写员工总数`：符合R2.1条件的员工数量。
        - `实际填写日志的员工数`：在统计周期内，至少提交过1篇日志的员工数量。
    3.  **按时提交率计算规则 (R2.3)**：
        - `按时提交率 = (规定截止时间前提交日志的员工数 / 应填写员工总数) * 100%`
        - `规定截止时间`：系统配置，默认为日志所属日期的23:59:59。
    4.  **平均填写时长计算规则 (R2.4)**：
        - `平均填写时长 = ∑(每位员工单次日志的【提交时间戳 - 开始编辑时间戳】) / 总填写次数`
        - 单位：分钟，保留一位小数。
        - IF 无任何日志记录 THEN 显示“--”。
    5.  **数据更新规则 (R2.5)**：所有指标数据于每日凌晨2:00通过定时任务计算更新（T+1）。
- **输出**：在页面展示3张指标卡片，内容为：员工填写率、按时提交率、平均填写时长。
- **验收标准**：
    1.  根据查询条件，正确显示三个核心指标的值。
    2.  无数据时，平均填写时长显示为“--”。

### 功能点3：明细数据列表与排序
- **输入**：来自功能点1的查询条件。
- **规则**：
    1.  **列表内容规则 (R3.1)**：
        - IF 查询维度为“部门” THEN 列表列包括：部门名称、员工总数、已填写人数、填写率、按时提交人数、按时提交率、平均填写时长（部门平均值）。
        - IF 查询维度为“员工” THEN 列表列包括：员工姓名、所属部门、应填写天数、已填写天数、填写率、按时提交天数、按时提交率、平均填写时长（个人平均值）。
    2.  **排序规则 (R3.2)**：用户可点击表头“填写率”、“按时提交天数”等数值型列进行升序/降序排序。
    3.  **分页规则 (R3.3)**：列表默认按“部门名称”或“员工姓名”拼音升序排列，每页显示20条数据，支持分页。
    4.  **交互规则 (R3.4)**：
        - IF 查询维度为“员工” AND 用户点击某员工姓名 THEN 跳转至该员工的个人日志历史页面（假设页面已存在）。
- **输出**：展示符合查询条件的数据明细表格。
- **验收标准**：
    1.  列表内容与当前查询维度匹配。
    2.  点击排序，列表数据按对应列正确排序。
    3.  数据量超过20条时，分页功能正常。

### 功能点4：数据导出
- **输入**：当前页面功能点1的所有查询条件。
- **规则**：
    1.  **导出触发规则 (R4.1)**：IF 用户点击“导出Excel”按钮 THEN 系统开始准备数据。
    2.  **文件命名规则 (R4.2)**：`日志填写统计_维度_部门名或员工名_统计周期_导出时间.xlsx`（例：`日志填写统计_部门_市场部_20260301-20260331_20260401103025.xlsx`）。
    3.  **文件内容规则 (R4.3)**：导出的Excel文件内容与当前“数据明细区”的表格列完全一致。
- **输出**：一个可供下载的Excel文件。
- **验收标准**：
    1.  点击导出后，成功下载Excel文件。
    2.  文件名和内容符合规则。

## 4. 规则补充
- **全局业务规则**：
    - **G1. 无数据处理**：IF 查询条件下无任何符合R2.1条件的员工或日志记录 THEN 数据概览区显示“--”，明细列表区显示“当前查询条件下暂无数据”。
    - **G2. 权限校验**：IF 用户尝试查询其无权限访问的部门数据 THEN 系统提示“您暂无权限查看该部门数据”，并阻止查询。
    - **G3. 导出失败处理**：IF 数据导出过程失败 THEN 系统提示“导出失败，请稍后重试或联系系统管理员”。

## 5. 示例（必须）
### 正常：
**场景**：销售部张经理查看本部门“昨日”日志统计。
- **Input**:
    - 维度：部门
    - 部门：销售部
    - 周期：昨日
    - 点击“查询”
  **Output**:
    - **数据概览区**显示：
        - 员工填写率：94.7%
        - 按时提交率：89.5%
        - 平均填写时长：8.3分钟
    - **数据明细区**显示销售部昨日所有员工的填写明细列表。

### 异常：
**场景1**：张经理尝试查询无权访问的技术部数据。
- **Input**:
    - 维度：部门
    - 部门：技术部 (此部门不在其管辖树内)
    - 点击“查询”
  **Output**:
    - 系统弹出提示：“您暂无权限查看该部门数据”，查询被中断，下方数据区域不刷新。

**场景2**：HR小王查询一个全新部门“上月”数据，该部门上月无任何正式员工。
- **Input**:
    - 维度：部门
    - 部门：新业务拓展部
    - 周期：上月
    - 点击“查询”
  **Output**:
    - **数据概览区**显示：
        - 员工填写率：--
        - 按时提交率：--
        - 平均填写时长：--
    - **数据明细区**显示文字：“当前查询条件下暂无数据”。
